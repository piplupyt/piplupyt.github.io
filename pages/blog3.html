Advanced TypeScript Tips

2024-01-20 Technology Reading time: 8 min

TypeScript extends JavaScript with a powerful type system. In this article, I share practical advanced techniques that help teams build safer and more maintainable applications.

Advanced Generic Patterns

Generics let you write reusable logic without losing type safety.

1. Conditional Types

Conditional types select a type based on a rule:

type NonNullable<T> = T extends null | undefined ? never : T;

type Example1 = NonNullable<string | null>; // string
type Example2 = NonNullable<number | undefined>; // number

2. Mapped Types

Mapped types transform an existing type into a new one:

type Partial<T> = {
    [P in keyof T]?: T[P];
};

interface User {
    id: number;
    name: string;
    email: string;
}

type PartialUser = Partial<User>;
// { id?: number; name?: string; email?: string; }

Advanced Type Operations

1. Template Literal Types

Template literal types allow type-safe string patterns.

type EventName<T extends string> = `on${Capitalize<T>}`;

type MouseEvent = EventName<'click'>; // 'onClick'
type KeyboardEvent = EventName<'keydown'>; // 'onKeydown'

2. Recursive Types

Recursive types are useful for nested structures like JSON: 

type JSONValue =
    | string
    | number
    | boolean
    | null
    | JSONValue[]
    | { [key: string]: JSONValue };

const data: JSONValue = {
    name: "r14921a10",
    age: 25,
    hobbies: ["coding", "reading"],
    address: {
        city: "Taipei",
        country: "Taiwan"
    }
};

Decorators

Decorators can wrap methods and add behavior such as logging, caching, or access control. 

function Log(target: any, propertyName: string, descriptor: PropertyDescriptor) {
    const method = descriptor.value;

    descriptor.value = function (...args: any[]) {
        console.log(`Calling ${propertyName} with args:`, args);
        return method.apply(this, args);
    };
}

class Calculator {
    @Log
    add(a: number, b: number): number {
        return a + b;
    }
}

Modules and Namespaces

1. ES6 Modules

TypeScript works naturally with ES modules:

// math.ts
export const add = (a: number, b: number): number => a + b;
export const subtract = (a: number, b: number): number => a - b;

// main.ts
import { add, subtract } from './math';
import * as MathUtils from './math';

2. Namespaces

Namespaces are another way to organize related code:

namespace Geometry {
    export interface Point {
        x: number;
        y: number;
    }

    export class Circle {
        constructor(public center: Point, public radius: number) {}

        area(): number {
            return Math.PI * this.radius * this.radius;
        }
    }
}

// use namespace
const point: Geometry.Point = { x: 0, y: 0 };
const circle = new Geometry.Circle(point, 5);

Useful Utility Types

Built-in utility types reduce repetitive type definitions:

  • Pick<T, K>: select properties K from T
  • Omit<T, K>: remove properties K from T
  • Record<K, V>: map keys K to values V
  • Exclude<T, U>: remove members assignable to U
  • Extract<T, U>: keep members assignable to U 
interface User {
    id: number;
    name: string;
    email: string;
    password: string;
}

// remove sensitive field
type PublicUser = Omit<User, 'password'>;

// keep only selected fields
type UserSummary = Pick<User, 'id' | 'name'>;

Best Practices

1. Enable Strict Type Checking

Use strict compiler options in tsconfig.json: 

{
    "compilerOptions": {
        "strict": true,
        "noImplicitAny": true,
        "strictNullChecks": true,
        "strictFunctionTypes": true
    }
}

2. Let Inference Work for You

Rely on inference when possible and annotate intentionally: 

// good practice
const users = ['alice', 'bob', 'charlie'];

// explicit only when needed
const namedUsers: string[] = ['alice', 'bob', 'charlie'];

3. Prefer Unions over any

Avoid any and model real constraints with union types: 

// avoid
function processData(data: any): any {
    return data;
}

// better
function processData(data: string | number | boolean): string | number | boolean {
    return data;
}
